## 🎬 수업 시연 포인트

### 실습#1 — BrowserRouter 감싸지 않고 실행 → 에러 시연
```
Uncaught Error: useRoutes() may be used only in the context of a <Router> component.
```
학생들이 이 에러를 나중에 마주쳤을 때 원인을 기억하도록 **일부러 에러를 보여주고** 고치는 과정을 시연하세요.

### 실습#2 — Network 탭 vs Link 차이 실연
1. DevTools Network 탭 열기
2. `<a>` 버전 먼저 작성 → 링크 클릭할 때마다 `Doc` 재요청 쌓임을 보여줌
3. `<Link>`로 교체 → Doc 요청이 **멈춤**을 보여줌
4. **이 차이가 SPA의 본질**임을 강조

### 실습#3 — NavLink `end` 누락 버그 실연
홈 링크에 `end`를 빼고 모든 페이지에서 홈이 활성화되는 것을 보여준 뒤 `end`로 해결. "왜 이게 기본값이 아닌지"에 대한 질문을 유도하세요.

### 실습#4 — 무한 루프 버그 실연
```jsx
function Home() {
  const navigate = useNavigate()
  navigate('/movies')  // 렌더 시점
  return <h1>홈</h1>
}
```
브라우저가 멈추는 것을 보여주고, 왜 이런 일이 생기는지 학생들에게 질문하세요.

### 실습#5 — 사이드바/본문 교체 실연
`/movies/drama` → `/movies/action` 전환 시 **사이드바가 사라졌다 다시 그려지는 게 아니라** 본문만 바뀐다는 것을 강조. React DevTools의 Profiler로 실제 리렌더를 보여주면 더 좋습니다.

### 실습#7 — 뒤로가기 무한 루프 실연
`<Navigate to="/login">` (replace 없음)을 넣고, 로그아웃 상태에서 `/mypage`로 간 뒤 뒤로가기를 눌러 무한 루프를 재현. `replace`를 추가해 해결.

### 실습#8 — URL 공유 실연
검색 결과 URL을 복사해 **다른 브라우저/시크릿 탭**에 붙여넣어 같은 결과가 나오는 것을 보여줌. "왜 useState로 검색어를 관리하면 이게 불가능한지" 질문으로 연결.

## 🌿 solutions 브랜치 관리

정답은 별도 브랜치로 관리해 학생의 답안 조기 열람 부담을 낮춥니다.

### 초기 세팅 (강사가 1회)

```bash
# 초기 상태 커밋 (메인 브랜치)
git add .
git commit -m "chore: 실습 스타터 베이스라인"

# solutions 브랜치 생성
git checkout -b solutions

# 각 실습 완료 상태를 순차적으로 구현하며 커밋
# 실습#1 구현 (pages/Home, Movies, About + App.jsx Routes + main.jsx BrowserRouter)
git add src/
git commit -m "solution 실습#1: Routes 3개 매핑"
git tag solution-실습01

# 실습#2 구현 (Header.jsx Link 교체 + App.jsx Header 배치)
git commit -am "solution 실습#2: Link 전환"
git tag solution-실습02

# ... 실습#8까지 반복
```

### 학생이 막혔을 때

```bash
# 특정 파일만 정답으로 교체
git fetch origin solutions
git checkout solutions -- src/pages/MovieDetail.jsx

# 특정 실습 시점으로 src/ 전체를 롤백
git fetch --tags origin
git checkout solution-실습05 -- src/
```

### 원격 저장소 배포 전략

**옵션 A — 단일 레포, 브랜치 분리 (권장)**
- 학생 기본 clone: main 브랜치만
- solutions 브랜치는 `git fetch origin solutions`로 필요 시 가져옴

**옵션 B — 레포 2개 분리**
- `react-router-practice`: 학생용 (main만)
- `react-router-practice-solutions`: 강사 열람용

학생 규모가 크거나 정답 열람 제어가 엄격하다면 옵션 B가 안전합니다.
